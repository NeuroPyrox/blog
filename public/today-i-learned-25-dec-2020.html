Today I Learned

25 Dec 2020

  • If you don't protect login forms against CSRF, then an attacker can switch your session to a different account and steal sensitive information that you submit
  • The synchronizer token pattern is more secure than double submit cookies because cookies can be overwritten by subdomains, allowing CSRF
  • Server-side state for non-authenticated sessions makes you more vulnerable to DOS attacks
  • Helmet is a small JavaScript library for setting http headers
  • The X-Frame-Options header can protect against clickjacking by preventing your site from being contained in a frame
  • The SameSite header can prevent Iframes from accessing cookies
  • If you store session IDs in URLs, then an attacker can inject their own session ID through a malicious link
  • Authentication and tamper detection are the 2 ways to detect MITM attacks
  • TLS provides a way to authenticate public keys via a trusted 3rd party, somewhat preventing MITM attacks
  • Latency examination can uncover MITM in some situations, especially with long calculations that take several seconds like hash functions
  • Punycode is a unicode representation only using ascii characters
  • Punycode can be used to prevent url spoofing by protecting against similar-looking unicode characters
  • SSL stripping is a MITM attack which downgrades a connection into an unencrypted one
  • Not practical, but quantum computing can theoretically mitigate MITM through the no-cloning theorem
  • A MITM can write cookies via HTTP redirects
  • HSTS can be used to prevent a MITM from writing cookies
