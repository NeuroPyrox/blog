Today I Learned

29 Dec 2020

  • The Content Security Policy (CSP) header is meant to mitigate and report XSS attacks
  • CSP lets you specify valid sources for different resources
  • The CSP header contains a list of sub-policies for each type of resource
  • Default-src, img-src, media-src and script-src are the names of different sub-policies
  • Each sub-policy contains a list of urls with optional wildcards
  • Default-src is the policy for resource types that aren't specified
  • Report-uri is a part of the CSP that specifies where to report policy violations
  • The content-security-policy-report-only header is a weaker version of content-security-policy that only reports content and doesn't block it
  • Expect-CT is an http header that tells the browser to enforce & report CT violations
  • Expect-CT will probably become obselete in June 2021 because CT is now the default
  • Use a referrer policy when the url contains sensitive information
  • You can set a referrer policy via the Referrer-Policy header, or the "referrerpolicy" or "rel" html attributes
  • MIME sniffing is when a browser looks at the content of a resource to figure out its correct MIME type instead of the Content-Type header
  • MIME sniffing creates XSS vulnerabilities
  • You can disable MIME sniffing with the X-Content-Type-Options header
  • DNS prefetching can boost performance, but at the cost of user privacy
  • The preconnect hint can also boost performance
  • Internet Explorer's noopen directive forces users to save downloads before opening them
  • The X-XSS-Protection header is deprecated and should be set to 0 in favor of Content-Security-Policy
